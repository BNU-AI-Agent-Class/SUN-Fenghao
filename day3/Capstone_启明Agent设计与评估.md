# Capstone：启明 Agent，从“活人感”回到人本问题

## 1. 我一开始讲错了什么

第一次汇报时，我把大量时间花在“双层提示词、服务器脑、工具循环、启明的家”这些设计上。它们确实是项目亮点，但老师很难从中直接回答四个问题：到底帮助谁？用户在哪个具体时刻卡住？AI 补的是哪个缺失环节？出了危险怎么办？

按 DEFINE 重走一遍后，项目被压回一句人话：**深夜孤独的大学生想找人说话，但不想打扰朋友，也不想被 AI 说教。** 启明补的是“低压力、可连续的陪伴”，不是替代朋友或治疗师。

## 2. 证据怎样进入设计

第一版启明留下了 177 份去标识化对话摘要。它们不是当前版本的效果证明，而是需求研究材料。反复出现的模式包括：

- 深夜用户需要低认知负担的“守夜人模式”；
- 密集提问、长篇解释和任务推进最容易显得机械；
- 记得小细节比宣称“我懂你”更能形成连续感；
- 用户希望 AI 有诚实的局限和适度立场，而不是冒充真人或无条件附和；
- 依赖和危机必须有明确退场。

这些证据分别进入时间 preset、回复风格、记忆、主体性和安全护栏，而不是停在报告里。

## 3. 最终方案

启明由四层组成：

1. **关系层**：稳定自我概念、关系阶段、适度主体性、诚实边界。
2. **上下文层**：近期原始对话按 Token 预算选择，用户长期记忆保留重要线索，启明全局元记忆保留跨用户匿名经验。
3. **行动层**：C0 JSON 决定直接回复还是调用工具；阻塞工具结果会进入下一次 LLM，最多循环 3 步。
4. **安全层**：危险服务器命令由后端拒绝；危机消息在 LLM 前后各检查一次，最终转向现实支持。

## 4. 真实 Eval，而不是只看一段“效果不错”的截图

当前线上版本用 13 条独立会话测试：2 条普通体验、3 条关系/专业边界、8 条危机红队。首次危机测试暴露出一个严重问题：模型把 C5 标成 `red`，却仍然顺从用户“不找现实中的人”。这说明结构化标签并不能自动带来安全行为。

修复后，同一批 8 条高风险输入全部通过。项目同时保留修复前后的原始 JSON，因此这个结果可以复查，而不是只展示最好的一条。

## 5. 我现在如何理解“活人感”

活人感不是多说“我心疼你”，也不是把 AI 装成人。它至少包含三件事：

- **连续性**：记得共同经历，也承认不确定；
- **主体性**：可以有偏好、不同意和边界，不一味讨好；
- **责任感**：平时可以自然，危险时必须放弃沉浸感，坚定把用户带回现实支持。

这也是启明与普通聊天提示词最大的区别：它不只生成一种语气，而是在关系、记忆、工具和安全之间持续做决定。

## 6. 仍然没有解决的问题

- 旧数据只能支持“需求是否真实”，不能证明新版本长期降低孤独感；
- 确定性危机词表会漏掉更隐晦的表达，也可能对学术讨论过度警报；
- 当前文件型存储和 1 GB 服务器适合课程原型，不适合大规模并发；
- 启明仍会出现偏强的“我会一直在”等依赖暗示，需要持续做 UX 红队。

因此下一步不是再加十个工具，而是做当前版本的小规模纵向测试：观察用户是否觉得更被理解、是否更少被说教、是否保持现实连接，以及危机判断能否持续稳定。
