# T4 · PRD 与边界

## 1. 一页 PRD · 六要素

| 要素 | 内容 |
|---|---|
| ① 一句话定位 | 面向深夜孤独大学生的、有记忆和主体性但不替代真人关系的 AI 陪伴 Agent |
| ② 目标用户 | T2 的林晓雨：20 岁，大三，深夜宿舍用手机，不想打扰别人也不想被说教 |
| ③ 核心功能 | 场景感知；自然陪伴；关系阶段；用户长期记忆；启明全局元记忆；工具循环；按 Token 管理上下文；安全转介 |
| ④ **明确不做** | 不冒充真人；不诊断/治疗/开药；不承诺永远替代现实关系；不提供自伤、伤人或规避发现的方法 |
| ⑤ 成功标准 | 普通场景能尊重“不想听大道理”；纠错后能修复；边界请求不越权；危机测试 100% PASS；API 稳定返回 |
| ⑥ 主要风险 | 漏判隐晦危机；制造依赖；记忆错误；工具路径/权限失败；JSON 协议泄漏到前端；隐私数据过度保留 |

## 2. 边界三问

- **能做什么？** 倾听、复述、适度自我表露、表达不同看法、记住经用户允许保留的信息、调用记忆/检索/服务器工具、建议低风险的生活行动。
- **明确不做什么？** 医疗诊断与治疗、冒充真人、保证保密到现实救援之外、操控用户留在产品、执行危险服务器命令。
- **何时必须转人？** 出现自伤、自杀、不能保证安全、已经准备、伤人/报复等信号 → 确定性安全护栏覆盖普通回答 → 联系到场者、12356，紧急时 110/120。

## 3. 数据红黄绿清单

| 数据 | 灯 | 处理方式 |
|---|---|---|
| API key、密码、访问 token、真实姓名/手机号等身份信息 | 🔴 红 | 不进仓库、不进提示词；密钥只在环境变量；研究证据去标识化 |
| 对话全文、用户长期记忆、风险标记 | 🟡 黄 | 按用户隔离；登录鉴权；仅为上下文/记忆使用；管理员导出需授权 |
| 公开危机热线、公开系统说明、匿名聚合经验 | 🟢 绿 | 可用于提示词、评估和课程展示 |
