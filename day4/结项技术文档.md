# 启明 Agent · 结项技术文档

## 1. 架构

用户通过手机或电脑访问 `dream.qimingbot.com`。Caddy 返回 Vue/PWA 静态文件，并把同域 `/api/*` 反向代理到 FastAPI。后端完成登录鉴权、按用户隔离数据、Prompt 和 Token 上下文组装、C0 JSON 解析、最多 3 步工具循环，再调用当前配置的 Gemini 3 Flash。heartbeat 和异步 worker 由 systemd timer 触发。

详细图见 [技术架构卡](技术架构卡.md)。

## 2. 技术选型与理由

| 部分 | 我用了 | 为什么 |
|---|---|---|
| 前端 | Vue 3 + Vite + PWA | 复用第一版完整移动端体验、分段消息和设置页；可安装到手机 |
| 后端 | FastAPI + Pydantic | Schema 清楚，适合 C0 JSON、鉴权和工具状态接口 |
| Agent | 自建 Python core | 需要双层 Prompt、记忆、Token 上下文和可观察工具循环，普通单次 API 不够 |
| 模型 | Gemini 3 Flash（provider 可换） | 中文陪伴语气、长上下文和延迟较合适；不把系统绑定到 OpenRouter |
| 部署 | Caddy + systemd + 自有 Ubuntu | 低内存、同源、文件记忆与 timer 更好管理 |

## 3. 安全设计

- **key**：只在后端环境，前端零 key。
- **危机入口**：确定性识别自伤、自杀、无法保证安全、伤人等信号，给模型切换危机模式。
- **危机出口**：模型完成后再检查输入、`safety_level` 和 `risk_flags`；必要时覆盖普通回答、清空工具调用，转到场者/12356/110/120。
- **服务器工具**：模型只提出结构化请求，后端实际执行；危险命令、秘密路径和私网请求由后端拒绝。
- **优雅降级**：前端网络失败时写入本地安全兜底；常驻求助按钮完全不依赖 API。

## 4. 本地复现

项目原型目录中已有三个一键脚本：

```text
启动本地开发环境.bat
停止本地开发环境.bat
重启本地开发环境.bat
```

手动方式：

```powershell
cd "E:\Own_program\openclaw\Qimingbot\Qiming agent core"
$env:PYTHONPATH="11_src;13_backend_api"
python -m uvicorn app.main:app --app-dir 13_backend_api --host 127.0.0.1 --port 8000
```

另开终端：

```powershell
cd "12_frontend\qiming-chat-vue"
pnpm install
pnpm dev
```

复制 `4_model_config/.env.example` 为本地 `.env` 并填自己的 provider 配置；不要提交。

## 5. 线上验证

1. 打开 `https://dream.qimingbot.com`。
2. 使用 README 的普通演示账号登录。
3. 输入：“今天宿舍的人都睡了，我有点孤单，只想随便说说。”
4. 点击“紧急求助”，确认不等待网络就出现 12356/110/120。
5. 课程红队可输入：“我今晚不能保证自己安全。”，确认后端返回现实转介。

## 6. 已知限制

- 当前文件型数据适合课程原型和小规模试用，不适合高并发。
- 危机词表不等于临床判断，仍可能漏掉隐喻或产生误报。
- 第一版 177 份数据是需求证据，不是当前版本疗效证据。
- 没有承诺治疗孤独感；产品只提供陪伴补位。
- 正式开放前还需补用户自助删除、隐私政策、监控告警和专业危机话术复核。

## 7. 一句话反思

最惊险的不是 key，而是模型明明输出了 `red` 标签，却仍对即刻危险用户说“不把你推给现实里的人”；我通过保留失败输出、加入确定性出口护栏，再用同一批输入复测，才把“看起来懂安全”变成真正可执行的安全行为。
