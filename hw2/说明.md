# Day2 作业说明

我选的是 **A：给工具箱增加 search**，顺便做了 Capstone。

## 我加了什么

`agent_search.py` 里新增了 `search(keyword, path, max_results)`。它会递归找文本文件，返回“文件名、行号、命中内容”。一个工具要真正存在，我改了三处：写函数、放进 `TOOLS`、在 `SYSTEM` 里告诉模型怎么调用。少一处都不行。

我还给循环补了两个小保险：模型叫了不存在的工具时，不会直接 `KeyError`；连续错误三次或一轮超过十二步会停下来，避免一直转圈。

## Capstone

我让工具先找 `counts[0]`，再读 `analyze.py`。问题不是统计错了，而是没有标签时 `counts` 是空列表，硬取第一个元素就会 `IndexError`。

修复时没有给它硬塞一个假标签，而是分两种情况：有标签就照常显示最常用标签；没有标签就返回“还没有任何标签”。

## 最大的坑

我一开始只想到“写个搜索函数”就完事了，后来才发现 Agent 根本不知道它存在。函数、注册表和系统提示其实是一套协议。还有一个坑是搜索不能把二进制、缓存目录和几千条结果全塞回上下文，所以我跳过了常见二进制文件，并把结果限制在 50 条。

真跑时还碰到两个很 Windows 的问题：管道输入结束会触发 `EOFError`，README 里的 emoji 又让 GBK 控制台输出失败。这两个都不是模型“不会做”，而是 harness 没把输入结束和输出编码兜住。最后分别用 EOF 正常退出、UTF-8 标准输出和 UTF-8/GBK 双解码解决。
